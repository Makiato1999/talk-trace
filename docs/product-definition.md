# TalkTrace 产品定义

## 一句话定义

TalkTrace 将技术演讲转化为有证据支持的 Engineering Notes。

它不是普通的演讲摘要工具。普通摘要只是压缩内容，TalkTrace 更关注：

- 演讲在讲什么问题；
- 核心观点和技术原理是什么；
- 演讲者如何解释和论证；
- 有哪些示例、实践建议和限制；
- 笔记内容来自演讲的哪个位置。

## 目标用户

主要面向观看技术演讲并希望沉淀知识的工程师和技术学习者。

典型场景包括：

- 技术大会演讲；
- 课程和公开课；
- 架构或框架分享；
- 技术访谈与讲座；
- 内部技术分享。

## 基本流程

```text
导入技术演讲的 transcript、字幕或其他材料
  -> AI 生成候选笔记
  -> 用户沿证据回看演讲
  -> 检查和修改笔记
  -> 保存或导出 Engineering Note
```

## 产品核心

### 1. 笔记不是 transcript 的缩写

演讲按照时间展开，Engineering Note 应按照知识结构组织，例如问题、观点、原理、示例和实践建议。

### 2. AI 生成的是候选内容

AI 可以总结和修订，但输出仍是候选笔记，用户可以检查和修改。

### 3. 重要内容可以回到原文

核心观点、技术解释和实践建议等重要内容，应尽量关联 transcript 时间段、字幕片段或 slide。

### 4. 笔记可以演进

用户可以继续编辑或重新生成笔记。需要时保留版本，避免修改过程丢失。

## MVP

第一版只需要做好：

- 导入 transcript 或字幕；
- 生成结构化工程笔记；
- 展示重要内容对应的演讲证据；
- 允许用户修改笔记；
- 保存笔记；
- 导出 Markdown。

第一版暂不处理：

- 音视频下载与自动转写；
- 自动操作 GitHub、Jira 等外部系统；
- 复杂的多人协作；
- 企业级知识图谱；
- Skills 系统。

## 三类设计

```text
Engineering Note Protocol
  定义笔记长什么样，以及内容如何组合

Pipeline
  定义 AI 如何理解演讲、生成、检查和修订笔记

Schema 与数据库
  定义演讲、笔记、证据和版本如何保存
```

三者相关，但不应混为一份设计。
